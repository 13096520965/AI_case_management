# 案件详情页面提醒消息展示

## 功能说明

在案件详情页面的基本信息卡片下方添加了提醒消息展示功能。当该案件有相关的提醒消息时，会在一个单独的卡片中展示。

## 实现细节

### 1. 数据获取逻辑

在 `CaseDetail.vue` 中添加了 `fetchCaseNotifications()` 方法，该方法：

1. 获取所有通知列表
2. 获取所有提醒规则
3. 获取该案件的所有流程节点
4. 过滤出与该案件相关的通知
5. 对每个通知，获取其关联的节点和案件信息
6. 只显示属于当前案件的通知

### 2. 提醒消息卡片

提醒消息卡片包含以下信息：

- **节点名称**：关联的流程节点名称
- **提醒类型**：节点到期、节点超期、费用支付等
- **提醒内容**：通知的具体内容
- **截止日期**：节点的截止日期
- **提醒时间**：通知的创建时间

### 3. 样式设计

- 卡片左侧有红色边框，表示重要性
- 每个通知项有不同的颜色标识（根据提醒类型）
- 提醒类型使用 Tag 组件显示，便于快速识别

## 提醒类型映射

| 提醒类型 | 标签类型 | 说明 |
|---------|---------|------|
| deadline | warning | 节点到期 |
| overdue | danger | 节点超期 |
| payment | primary | 费用支付 |
| task | success | 协作任务 |
| system | info | 系统通知 |

## 显示条件

- 只有当案件有相关提醒消息时，才会显示提醒消息卡片
- 如果没有提醒消息，卡片不会显示

## 示例场景

对于内部编号为 `AN202511000008` 的案件：
- 如果"送达起诉状副本"节点的截止日期是 `2025-11-26`
- 提醒规则中"节点到期"的提醒是"前三天"
- 则在 `2025-11-23` 及之后，该案件详情页面会显示相应的提醒消息

## 相关 API

- `notificationApi.getNotifications()` - 获取所有通知
- `notificationRuleApi.getRules()` - 获取提醒规则
- `processNodeApi.getNodesByCaseId(caseId)` - 获取案件的流程节点
- `request.get('/nodes/{id}')` - 获取节点详情
- `request.get('/cases/{id}')` - 获取案件详情
